- **Command Run:** `uvicorn app/main.py`
- **Context:** Attempted to start the Uvicorn server by providing the file path to `main.py`.
- **Result:** Failed with the following error:
  `ERROR: Error loading ASGI app. Import string "app/main.py" must be in format "<module>:<attribute>".`
  This occurred because Uvicorn expects a Python import string (e.g., `module:app_instance`) rather than a direct file path.

### Step 5: Navigating to the App Directory
- **Command Run:** `cd app`
- **Context:** Changed directory into the `app` folder where the `main.py` file is explicitly located, making it easier to reference the module.

### Step 6: Successfully Starting the Server
- **Command Run:** `uvicorn main:app --reload`
- **Context:** Ran the Uvicorn server using the correct import string format:
  - `main`: Refers to the `main.py` file.
  - `app`: Refers to the `FastAPI()` application instance created inside `main.py`.
  - `--reload`: Enables hot-reloading so the server automatically restarts whenever code changes are saved.
- **Result:** The FastAPI development server started successfully and is currently running.
- **Testing Query:** `http://127.0.0.1:8000/` (Returns `{"message": "Hello World"}`)
